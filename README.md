# CRUD一覧ジェネレーター（PWA）

Forguncyの `.fgcp` を渡すと、**ページ別CRUD一覧Excel**（4シート・色分け）を生成してダウンロードできるブラウザアプリです。
`generate_crud.py` のロジックをそのままJSへ移植しています。

## 最大の特長：端末内で完結
- `.fgcp`（プロジェクト一式）もForguncyドキュメントも、**どこにも送信・保存しません**。
- 解凍（JSZip）・CRUD抽出・Excel生成（ExcelJS）は、すべてブラウザの中だけで実行されます。
- サーバー不要。オフラインでも動作します（PWA）。

| `forguncy-crud-pwa/`（zip） | GitHub Pages公開用。インストール・オフライン対応。 |

## 使い方
1. ページを開く（単一版はダブルクリック／タップ）。
2. **①** に `.fgcp` を選択（必須）。スマホは枠を**タップ**（ドラッグ&ドロップはPCのみ）。コピー推奨（元ファイルは変更しません）。
3. **②** に Forguncyの「ドキュメント生成」Excel を選択（任意）。付けるとページにNo.が入ります。
4. 「CRUD一覧を生成」→「Excelをダウンロード」。`ページ別CRUD一覧_YYYYMMDD.xlsx` が保存されます。

## 対応診断（バージョン互換チェック）
「このファイルで使えるか診断する」を押すと、`.fgcp` を解析して次を表示します。
- ページJSONの検出件数・パース成否・`AttachInfos` の有無・末尾署名の有無
- **JSONフォルダ構成**（`PAGE` / `PAGES` / `TABLES` 等）と、ページ・テーブルJSONの最上位キー
- 出てきたコマンド種別（**認識できるもの／未対応のもの**）とセル種別
- `UpdateType` の内訳（「(未指定)」が多いと C↔U の取り違えに注意）
- 総合判定：**○ 使える／△ 一部漏れの可能性／× 構造が異なる可能性**

> ページのフォルダ名は `PAGE` / `PAGES` / `Page` / `Pages` のいずれでも自動で拾います。`TABLES` フォルダがあればテーブル定義名を読み、どのページからも操作されない**未使用テーブル候補**を関連図に出します（テーブル名とページ参照名が一致する場合）。

手持ちのForguncyの版で使えるかを、実データで即判定できます。`×`や未対応コマンドが出たら、その内容に合わせて `crud-core.js` の抽出ロジックを調整します。

### 古い版の `.fgcp` を使うとき
版ごとに `.fgcp` の内部形式が異なり、新しいBuilderで開くと変換（アップグレード）されます。古い版で `×` 判定になる場合は、いったん現行のForguncy Builderで開いて保存し直し、変換後の `.fgcp` を渡すと確実です（**変換は元に戻せないので必ずコピーで作業**）。

## 出力シート
| シート | 内容 |
|---|---|
| 統計サマリ | 全体統計＋業務領域別ページ数 |
| CRUDマトリクス | 全ページのC/R/U/D・業務領域・注記（C緑・U黄・D橙、オートフィルタ・行固定付き） |
| テーブル別逆引き | テーブル→操作ページの逆引き |
| 削除ページ(要注意) | D操作を行うページのみ抽出 |

## GitHub Pages への配置
1. リポジトリ（例：`forguncy-crud`）を作り、**`forguncy-crud-pwa/` の中身を直下に**置く。
2. push 後、Settings → Pages → Branch を `main` / `(root)`。
3. `https://<ユーザー名>.github.io/forguncy-crud/` で公開。HTTPSなのでPWA（インストール・オフライン）が有効になります。
> パスは相対参照（`./`）なので、サブパス配信でもそのまま動きます。

## ローカルで確認する
Service Worker は `localhost` か HTTPS でのみ動きます（単一版は不要）。
```bash
cd forguncy-crud-pwa
python3 -m http.server 8000   # → http://localhost:8000/
```

## 更新したいとき
ファイルを変更したら、`sw.js` の `CACHE = 'crud-v2'` の番号を上げてください（`v3` など）。古いキャッシュが破棄され、新しい版が反映されます。

## ロジックの限界（`generate_crud.py` と同じ）
1. **サーバーサイドコマンド（SSC）のDB書込みは自動検出されない。** `crud-core.js` の `SSC_NOTES` の注記のみ。SSC追加ページは手で追記。
2. **`UpdateType` 未指定は「更新(U)」扱い。** add相当がType省略だと C が U に化けます（診断のUpdateType内訳で確認可）。
3. コマンドを日本語化→正規表現で抽出するため、テーブル名に `]` が含まれると壊れます。
4. ドロップダウンの DataSource を R に加算。クエリ名/数式が混じるとRに軽くノイズ。

> 注：業務領域の分類ロジックには、実在ページを判定するための照合キーワードが含まれます（表示ラベルではありません）。汎用化のため一般のForguncyプロジェクトでは多くが「M システム・補助」に分類されます。
